2. **Component Reference Only**

3. **File Structure**
- Follow **Google Android App architecture guidelines** for all files.
- Place Activities, Fragments, ViewModels, and other components according to recommended Android package structures.

4. **Performance**
- Ensure code is optimized for **excellent performance**.
- Avoid unnecessary UI inflation, memory leaks, or blocking operations on the main thread.
- Use modern Android development practices (e.g., Jetpack libraries, ViewModel, LiveData/Flow, Data Binding/Compose where applicable).

5. **Coding Practices**
- Use **Kotlin** for development.
- Ensure code is readable, maintainable, and modular.
- Apply proper MVVM/MVI architecture patterns as appropriate.
- Keep UI logic separate from business logic.

6. **AI Instructions**
- Always refer back to the reference project for **UI components and styling**.
